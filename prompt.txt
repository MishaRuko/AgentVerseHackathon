I want to change the frontend under the frontend directory to work with my project. It was adapted from the frontend for another project, protalab.
In this project, I will be visualising a graph and allowing the user to ask questions about the graph. The websocket communication protocol will work differently. This is how it's going to work:

At first, there will be nothing visible in the graph viewer area. The user will be able to send a message about what social area they'd like to learn about (this is a project that combines information from various sources to build a graph showing the relationship between dominating ideas). The message will be sent via the websocket as just text. We will then expect to receive a node datastructure, annotations, and a similarity matrix.

Explanation for the node datastructure: look in clustering.py. The cluster_and_summarise method will return a bunch of nodes. We will be connecting these. Note that we will not be displaying the ideas under each cluster at the moment.
Explanation for annotations: look in graph_builder.py. The build_cluster_graph function returns 3 things. The first thing will be used for other purposes. The group to explanations dictionary will contain a tuple of indices that will be for a group of nodes that are connected. The indices will correspond to the nodes in the graph datastrucutre. The explanations will be an annotation for the connection between the nodes in the tuple.
Explanation for the similarity matrix: the third thing returned from the function mentioned above. We can use the similarity values to understand the strength of the connections between nodes. Nodes that have 0 strength are unconnected and only nodes that are connected according to the annotation groupings are considered connected. Everything else is not considered connected.

We will receive these values in JSON format via the ws. The graph should then be displayed with the appropriate annotations and connections, the strength of which should be shown by their transparency/thickness. The size of the clusters should be proportional to the number of ideas under them.

Once the graph is displayed, the user will be able to send queries about the graph via the chat interface. This will be sent as text via the websocket. A response will be generated by the backend and sent via the websocket again. There will be no persistent state or full state retrieval endpoint in this project, just one volatile session. Note that the backend is not implemented yet.

Please make the necessary modifications to the frontend to interact with a stub backend. Please also make the frontend build a graph from the returned data as described above.
